# Mantri — V1 Specification

**Mac-only | Browser-based | Two buttons: Podcast + Chat**

V1 is a Mac-only assistant that runs locally. It connects to ProtonMail via Proton Bridge and exposes email capabilities as MCP tools. The UI is a simple browser page with two buttons.

## The Two Buttons

### Podcast Button — Passive Listening

Hit the Podcast button and Mantri generates a conversational podcast covering your **unread newsletters**. Two AI co-hosts discuss the content — what's interesting, what's new, what matters — in a natural, engaging format. You just listen.

- **Source**: Unread emails in the Newsletters folder only
- **Format**: Two-voice conversational podcast (ElevenLabs GenFM, "conversation" mode)
- **Length**: Adapts to volume, typically under 5 minutes
- **On completion**: Covered emails are marked as read
- **No interaction**: This is listen-only. You don't talk back to the podcast in V1.
- **Gist, not readout**: The hosts discuss the substance, they don't read email metadata or full bodies verbatim

### Chat Button — Interactive Agent

Hit the Chat button and you're in a voice conversation with a single agent. This is where you interact with your inbox. Ask questions, give commands, take actions.

**What you can ask:**
- "What are my unread emails?" → Agent gives a gist of each, not a metadata dump
- "Tell me more about that email from Sarah" → Agent reads more details on demand
- "Did I get anything from USPS?" → Agent checks, describes any images (e.g. package photos from Informed Delivery)
- "Reply to Sarah — tell her Thursday works" → Agent drafts, reads back, sends on confirmation
- "Send an email to Mike about the project deadline" → Compose from scratch by voice
- "Delete that AmEx ad" → Moves to trash with confirmation
- "Stop telling me about AmEx ads" → Updates PROFILE.md, learns for future sessions

**Key behaviors:**
- **Gist-first**: Never dumps full email contents unprompted. Starts with a one-line summary per email. You probe deeper when interested.
- **Image-aware**: For emails with images (like USPS Informed Delivery), describes what's in the photos ("looks like a small package from Amazon arriving today").
- **Learning**: Explicit instructions ("don't tell me about ads") update PROFILE.md immediately. Implicit patterns (you always skip certain senders) are learned over time.
- **Confirmation required**: Always reads back drafts before sending. Always confirms before deleting.

## What V1 Does NOT Include

- Interrupting the podcast to ask questions (V2 — unified experience)
- Mobile access (V3 — Cloudflare Tunnel / Tailscale)
- Scheduled/automated podcast generation (everything is on-demand)
- Calendar integration
- Deep content reading ("go read that blog post for me")
- Attachment content parsing (PDF reading, etc.)
- Multi-account support
- Proactive notifications
- Smart scheduling / delayed send

### Data Flow — Podcast Button

1. User clicks Podcast
2. MCP Server fetches unread emails from Newsletters folder via IMAP
3. Claude writes a two-host conversational podcast script, respecting PROFILE.md preferences (skip sections, noise filters)
4. Script → ElevenLabs GenFM Podcast API (`POST /v1/studio/podcasts`, mode: "conversation")
5. Audio streams/plays in the browser
6. Covered emails marked as read via IMAP

### Data Flow — Chat Button

1. User clicks Chat → opens ElevenLabs Conversational AI session (browser-based)
2. You speak → ElevenLabs STT transcribes
3. Transcript → Claude reasons + selects MCP tool
4. MCP tool call → Email MCP Server executes against Proton Bridge (IMAP/SMTP)
5. Result → Claude formulates gist-style response
6. Response → ElevenLabs TTS speaks back
7. If you give an explicit preference ("stop telling me about X") → Memory tool updates PROFILE.md

### Privacy Model

- Proton Bridge decrypts emails on localhost only
- Email content passes through Claude for processing (not stored)
- PROFILE.md lives on your local disk
- No email data stored on third-party servers
- MCP server runs entirely on your machine

## Technology Stack

| Layer | Technology | Why |

| Email access | Proton Bridge + imapflow + nodemailer | Official Proton decryption; imapflow is the best modern IMAP client for Node.js |

| Memory | PROFILE.md (flat file) | Simple, human-readable, trainable by voice |

## MCP Tool Specifications

### Email Reading Tools

| Tool | Parameters | Returns |
|------|-----------|---------|
| `get_unread` | `folder` (default: INBOX), `limit` | List of unread emails: id, sender, subject, date, one-line preview |
| `get_email` | `email_id` | Full email: sender, recipients, subject, date, body (text/HTML), attachment names, inline image descriptions |
| `get_newsletters` | `limit` | Unread emails from Newsletters folder with full body content (for podcast scripting) |
| `search_emails` | `query` (natural language), `folder`, `date_range` | Matching emails with relevance scores |

### Email Action Tools

| Tool | Parameters | Returns |
|------|-----------|---------|
| `mark_read` | `email_ids` (array) | Confirmation |
| `mark_unread` | `email_ids` (array) | Confirmation |
| `delete_email` | `email_ids` (array), `permanent` (default: false) | Confirmation; permanent=true skips trash |
| `move_email` | `email_ids`, `target_folder` | Confirmation |
| `send_email` | `to`, `subject`, `body`, `cc`, `bcc`, `reply_to_id` | Confirmation with message ID |

### Podcast Tools

| Tool | Parameters | Returns |
|------|-----------|---------|
| `generate_podcast` | `content` (newsletter digest text), `voice_a_id`, `voice_b_id` | Audio URL or file path for playback |

### Memory Tools

| Tool | Parameters | Returns |
|------|-----------|---------|
| `read_profile` | `section` (optional) | Current PROFILE.md contents or specific section |
| `update_profile` | `section`, `content`, `mode` (append/replace) | Confirmation |
| `log_interaction` | `action`, `context`, `outcome` | Logged for pattern learning |

## ElevenLabs Integration Details

### Chat Agent Setup (Conversational AI 2.0)

1. Create an ElevenLabs Conversational AI agent in the dashboard
2. Select Claude Sonnet as the LLM backend
3. Choose a voice from the library (or clone your own)
4. Add the Email MCP Server as an MCP tool integration
5. Configure tool permissions:
   - **Read tools** (get_unread, get_email, search_emails) → auto-approve
   - **Write tools** (send_email, delete_email, mark_read) → require confirmation for destructive actions
   - **Memory tools** (update_profile) → auto-approve
6. Write the system prompt:
   - Personality: helpful, concise, conversational
   - Always load PROFILE.md at session start
   - Gist-first: summarize emails in one line, expand only when asked
   - For image-containing emails: describe what's in the images
   - When user gives explicit preferences: update PROFILE.md immediately
   - Always read back drafts before sending
   - Always confirm before deleting

### Podcast Pipeline (GenFM)

The podcast is a separate pipeline, not part of the Conversational AI agent. It runs on-demand when the user clicks the Podcast button.

```
POST /v1/studio/podcasts

{
  "mode": "conversation",
  "source": {
    "type": "text",
    "content": "<two-host script generated by Claude from newsletter content>"
  },
  "quality_preset": "high",
  "duration_scale": "short"
}
```

The script generation step uses Claude (called directly from the MCP server, not via ElevenLabs) to:
1. Fetch unread newsletters via `get_newsletters`
2. Read PROFILE.md for preferences and noise filters
3. Write a natural two-host podcast script that discusses the content conversationally
4. Pass the script to GenFM for audio generation

LLM costs for GenFM podcast generation are currently covered by ElevenLabs. You pay only for audio credits.

### MCP Transport

ElevenLabs Conversational AI 2.0 supports MCP natively as a tool type (Feb 2026, SDK v2.36.0+). The local MCP server needs to be exposed so ElevenLabs can reach it.

Options:
- **ngrok** — quickest for development
- **Cloudflare Tunnel** — production-grade, free tier available
- **Tailscale Funnel** — zero-config if already using Tailscale

**Goal**: Programmatic access to ProtonMail inbox via MCP tools.

2. Configure connection to Proton Bridge (IMAP localhost:1143, SMTP localhost:587)

7. Test each tool via MCP Inspector

**Milestone**: Can read, search, and manage ProtonMail inbox programmatically. Memory persists.

### Phase 2: Chat Agent (Weekend 2)

1. Create ElevenLabs Conversational AI agent (Claude backend, chosen voice)
2. Expose local MCP server via ngrok
3. Add MCP server as tool integration in ElevenLabs dashboard
4. Configure tool permissions
5. Write agent system prompt (gist-first, PROFILE.md loading, image descriptions, learning behavior)
6. Build minimal browser UI with Chat button that opens the ElevenLabs widget
7. Test: ask about unread emails, probe deeper, send a reply, delete an email, teach a preference

**Milestone**: Can talk to your email assistant and it performs real actions on your inbox.

### Phase 3: Podcast (Weekend 3)

**Goal**: On-demand newsletter podcast.

1. Implement `get_newsletters` tool (fetch unread from Newsletters folder)
2. Build Claude prompt that writes a two-host conversational podcast script from newsletter content, respecting PROFILE.md
3. Implement `generate_podcast` tool (calls ElevenLabs GenFM API, conversation mode)
4. Add Podcast button to browser UI with audio player
5. Wire up: click Podcast → fetch newsletters → script → GenFM → play → mark as read
6. Test end-to-end with real newsletters

**Milestone**: Complete V1. On-demand podcast + interactive chat agent.
